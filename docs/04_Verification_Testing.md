# Step 4: Verification & Testing

## 1. 기능 테스트 (API & Logic)
- **정상 케이스:** 긍정/부정 문장 입력 시 정확한 결과 반환 확인.
- **예외 케이스:** 아주 짧은 단어, 빈 칸, 매우 긴 문장 입력 시 에러 핸들링.
- **DB 확인:** 분석 후 Supabase에 데이터가 정상적으로 쌓이는지 확인.

## 2. UI/UX 검증
- 모달창이 나타날 때 부드러운 Fade-in 효과가 있는지 확인.
- 모바일(iPhone 환경)에서 텍스트 입력창과 버튼이 깨지지 않는지 확인.
- Apple 디자인 시스템의 자간(`letter-spacing`)이 Headline에 적용되었는지 재검토.
